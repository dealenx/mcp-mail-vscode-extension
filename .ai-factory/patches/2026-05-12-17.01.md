# SMTP sendMail hangs indefinitely without timeout

**Date:** 2026-05-12 17:01
**Files:** src/mail/smtp-client.ts, src/mail/mailService.ts
**Severity:** high

## Problem

The `SMTPClient.sendMail` method calls `nodemailer.Transporter.sendMail` without any timeout. When the SMTP server is slow, unresponsive, or the connection silently stalls, the promise never resolves and the operation hangs forever. This causes the VS Code chat to show "Using Send Email" indefinitely.

## Root Cause

`nodemailer`'s `sendMail` does not have a built-in timeout for the overall send operation (only socket-level timeouts). If the server accepts the connection but never responds after `DATA`, the promise remains pending. There was no `Promise.race` or AbortController wrapping the call.

## Solution

Wrapped `this.transporter.sendMail` in `Promise.race` against a 30-second timeout promise in `SMTPClient.sendMail`. If the timeout fires first, an explicit `Error("Send mail timed out after 30000ms")` is thrown, which propagates to callers and causes the tool to fail fast instead of hanging.

Also added `[FIX]` prefixed logging in `MailService.sendEmail` and `replyToEmail` for easier troubleshooting of send successes in the future.

## Prevention

- Always race async network operations (SMTP, IMAP, HTTP) against a timeout promise.
- Consider a shared timeout helper for all network-bound methods.
- If logs are kept, keep `[FIX]` logs for a few iterations to verify the timeout works correctly in production.
