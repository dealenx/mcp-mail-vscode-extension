# Add remote mode indicator to sidebar and proxy logging

**Date:** 2026-05-28 20:25
**Files:** packages/vsix-extension/src/mailSidebar.ts, packages/vsix-extension/src/extension.ts, packages/service/src/index.ts, packages/vsix-extension/src/mail/remote-client.ts
**Severity:** low (feature, not bug)

## Problem

When using remote mode, there was no visual indication in VS Code about which send mode (local/remote) was active. Also, the service didn't explicitly log that requests were coming through as proxy requests.

## Root Cause

The sidebar always showed the same static items regardless of send mode. The service had Hono's logger middleware but no explicit "proxy" tagging.

## Solution

1. **VSIX sidebar**: Added a dynamic first item showing current send mode:
   - `🖥️ Локальный режим` (device-desktop icon) when local
   - `🌐 Удалённый режим (url)` (globe icon) when remote
   - Tooltip explains the mode and says "click to change settings"
   - Refreshes sidebar on config change (`onDidChangeConfiguration`)

2. **Service**: Added Hono middleware on `/api/*` that logs `[Proxy] METHOD /api/path from clientIp` for every request, making it clear in server logs that traffic is proxy traffic.

3. **RemoteClient**: Updated constructor log to include 🌐 emoji for visibility.

## Prevention

- When adding modes/states, always surface them in UI — hidden state confuses users
- Use middleware for cross-cutting concerns like request logging instead of per-route edits
